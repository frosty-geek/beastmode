# Phase 4: Complete (Merge & Cleanup)

## Overview

Guide completion of development work by merging back to main, cleaning up the worktree, and handing off to verification.

**Core principle:** Verify tests → Merge → Cleanup → Handoff.

**Announce at start:** "Completing implementation - merging and cleaning up."

## Step 1: Verify Tests

**Before merging, verify tests pass:**

```bash
# Run project's test suite
npm test / cargo test / pytest / go test ./...
```

**If tests fail:**
```
Tests failing (<N> failures). Must fix before merging:

[Show failures]

Cannot proceed until tests pass.
```

STOP. Do not proceed to merge.

**If tests pass:** Continue to Step 2.

## Step 2: Determine Base Branch

```bash
# Get the base branch (usually main or master)
base_branch=$(git symbolic-ref refs/remotes/origin/HEAD 2>/dev/null | sed 's@^refs/remotes/origin/@@')

# Fallback
if [ -z "$base_branch" ]; then
    base_branch="main"
fi
```

## Step 3: Present Options

Present exactly these 4 options:

```
Implementation complete. What would you like to do?

1. Merge back to <base-branch> locally
2. Push and create a Pull Request
3. Keep the branch as-is (I'll handle it later)
4. Discard this work

Which option?
```

**Use AskUserQuestion:**

```yaml
AskUserQuestion:
  question: "Implementation complete. How should I proceed?"
  header: "Complete"
  options:
    - label: "Merge locally"
      description: "Merge to <base-branch>, delete worktree, handoff to /verify"
    - label: "Create Pull Request"
      description: "Push branch, create PR, keep worktree"
    - label: "Keep as-is"
      description: "I'll handle merge later, keep worktree"
    - label: "Discard work"
      description: "Delete branch and worktree (requires confirmation)"
```

## Step 4: Execute Choice

### Option 1: Merge Locally (Recommended)

```bash
# Get current worktree path and branch
worktree_path=$(pwd)
feature_branch=$(git branch --show-current)

# Switch to base branch in main repo
cd "$(git rev-parse --show-toplevel)/.."
git checkout "$base_branch"

# Pull latest
git pull

# Merge feature branch
git merge "$feature_branch"

# Verify tests on merged result
<test command>

# If tests pass, cleanup
git worktree remove "$worktree_path"
git branch -d "$feature_branch"
```

**Then:** Handoff to /verify

### Option 2: Push and Create PR

```bash
# Push branch
git push -u origin "$feature_branch"

# Create PR
gh pr create --title "<title from plan>" --body "$(cat <<'EOF'
## Summary
<2-3 bullets from plan>

## Test Plan
- [ ] Tests pass locally
- [ ] Review implementation against plan

🤖 Generated by beastmode /implement
EOF
)"
```

Report PR URL. Keep worktree for potential follow-up.

**Then:** Wait for PR merge before cleanup

### Option 3: Keep As-Is

```
Keeping branch <branch-name>.
Worktree preserved at <worktree-path>.

When ready, you can:
- Resume with: cd <worktree-path>
- Merge manually: git checkout <base> && git merge <branch>
- Cleanup: git worktree remove <path> && git branch -d <branch>
```

**Do NOT cleanup worktree.**

### Option 4: Discard

**Require typed confirmation:**

```
This will permanently delete:
- Branch: <branch-name>
- All commits since branching
- Worktree at: <worktree-path>

Type 'discard' to confirm.
```

Wait for exact confirmation.

If confirmed:
```bash
cd "$(git rev-parse --show-toplevel)/.."
git checkout "$base_branch"
git worktree remove "$worktree_path" --force
git branch -D "$feature_branch"
```

## Step 5: Cleanup Worktree

**For Options 1 and 4 only:**

```bash
# Verify we're not in the worktree
cd "$(git rev-parse --show-toplevel)/.."

# Remove worktree
git worktree remove <worktree-path>

# Verify removal
git worktree list
```

**For Options 2 and 3:** Keep worktree.

## Step 6: Handoff to Verify

**For Option 1 (Merge Locally):**

```
Implementation merged to <base-branch>.
Worktree cleaned up.

Next step: Run /verify to complete the development workflow.
```

**Announce:** "I'm using the /verify skill to complete this work."
**REQUIRED SUB-SKILL:** Use beastmode:verify

## Quick Reference

| Option | Merge | Push | Keep Worktree | Cleanup Branch |
|--------|-------|------|---------------|----------------|
| 1. Merge locally | ✓ | - | - | ✓ |
| 2. Create PR | - | ✓ | ✓ | - |
| 3. Keep as-is | - | - | ✓ | - |
| 4. Discard | - | - | - | ✓ (force) |

## Common Mistakes

### Skipping test verification

- **Problem:** Merge broken code
- **Fix:** Always verify tests before offering options

### Automatic worktree cleanup for PR

- **Problem:** Remove worktree when might need it for PR feedback
- **Fix:** Only cleanup for Options 1 and 4

### No confirmation for discard

- **Problem:** Accidentally delete work
- **Fix:** Require typed "discard" confirmation

### Force-pushing

- **Problem:** Overwrite remote history
- **Fix:** Never force-push without explicit request

## Red Flags

**Never:**
- Merge with failing tests
- Delete work without confirmation
- Force-push without explicit request
- Leave orphaned worktrees

**Always:**
- Verify tests before merge
- Present all 4 options
- Get typed confirmation for discard
- Clean up only when appropriate
- Handoff to /verify after merge

## Exit Criteria

For each option:

**Option 1 (Merge):**
✓ Merged to base branch
✓ Tests pass on merged result
✓ Worktree removed
✓ Branch deleted
✓ Handed off to /verify

**Option 2 (PR):**
✓ Branch pushed
✓ PR created
✓ PR URL reported
✓ Worktree preserved

**Option 3 (Keep):**
✓ Instructions provided
✓ Worktree preserved

**Option 4 (Discard):**
✓ Confirmation received
✓ Worktree removed
✓ Branch deleted

**On success:** Workflow complete (or handed to /verify)
**On failure:** Stop and ask for help
