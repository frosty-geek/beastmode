## Problem Statement

Installing beastmode requires two disconnected steps — `claude plugin add` for skills, then cloning the repo and running `bun link` in `cli/` for the dashboard/orchestration CLI. The README barely documents this, lists no system requirements, and new users have no clear path from zero to working installation.

## Solution

Ship a single npm package (`beastmode`) that bundles both the plugin (skills, agents, hooks) and the CLI source. Users run `npx beastmode install` — one command that copies the plugin into Claude Code's marketplace directories, auto-installs bun if missing, runs `bun install` + `bun link` for the CLI, and verifies everything works. The README Install section gets rewritten with system requirements and the one-liner.

## User Stories

1. As a new user, I want to run a single command (`npx beastmode install`) to get the full beastmode system (plugin + CLI) installed, so that I don't need to discover and run multiple disconnected setup steps.

2. As a new user, I want the installer to auto-install bun if it's not on my machine, so that I don't need to manually install runtime dependencies before I can use beastmode.

3. As an existing user, I want to re-run `npx beastmode install` to update to the latest version, so that install and update are the same idempotent command.

4. As a user who wants to remove beastmode, I want to run `npx beastmode uninstall` to cleanly remove the plugin registration and CLI link while preserving my project-level `.beastmode/` data, so that uninstalling doesn't destroy my project knowledge.

5. As a new user, I want the installer to verify the installation by checking that `beastmode` and `claude` commands work, so that I know immediately if something went wrong.

6. As a prospective user reading the README, I want to see explicit system requirements (macOS, Node.js, Claude Code, Git, iTerm2) before I attempt installation, so that I don't waste time on an unsupported setup.

7. As a user on a system where the install fails, I want clear error messages telling me exactly what's missing or wrong, so that I can fix the issue without guessing.

## Implementation Decisions

- **npm package name**: `beastmode` (unscoped, confirmed available on npm)
- **Package contents**: `src/npx-cli/` (install command), `plugin/` (full plugin tree), `cli/` (dashboard/orchestration CLI source)
- **Install steps** (in order):
  1. Check prerequisites (Node.js >= 18, macOS, Claude Code installed)
  2. Detect existing installation — if found, overwrite with current version
  3. Auto-install bun if not found (using official install script)
  4. Copy `plugin/` to `~/.claude/plugins/marketplaces/bugroger/`
  5. Write version-specific cache to `~/.claude/plugins/cache/bugroger/beastmode/{version}/`
  6. Register marketplace in `~/.claude/plugins/known_marketplaces.json` (merge, don't replace)
  7. Register plugin in `~/.claude/plugins/installed_plugins.json` (merge, don't replace)
  8. Enable plugin in `~/.claude/settings.json` (merge, don't replace)
  9. Run `bun install --production` in cached `cli/` directory
  10. Run `bun link` in cached `cli/` directory to put `beastmode` on PATH
  11. Verify: run `beastmode --version` and `claude --version`
  12. Print summary (version installed, paths, next steps)
- **Uninstall steps**: Remove marketplace dir, remove plugin/marketplace entries from JSON configs, remove enabledPlugins entry, `bun unlink` the CLI. Leave project-level `.beastmode/` intact.
- **Versioning**: npm package version always matches `plugin.json` version (currently 0.99.0). CLI version syncs to match.
- **Output style**: Plain text console output. No interactive prompts, no progress spinners. Suitable for both terminal and piped/CI use.
- **Idempotency**: Re-running install detects existing installation and overwrites cleanly (clean-replace for plugin dirs, merge for JSON configs).
- **JSON config writes**: Atomic read-merge-write pattern to preserve unrelated settings.
- **`claude plugin add` path**: Removed for now. Single install path via npx only.
- **Project init**: Stays separate as `/beastmode init` inside Claude Code. Not part of the installer.

## Testing Decisions

- Integration test: run install on a clean machine mock (temp HOME dir), verify all files are created in correct locations with correct content
- Idempotency test: run install twice, verify second run succeeds and state is correct
- Uninstall test: run install then uninstall, verify cleanup is complete and project files are preserved
- Prerequisite check tests: mock missing Node.js, Claude Code, etc. and verify correct error messages
- JSON merge tests: verify existing settings.json content is preserved when adding plugin entries
- Bun auto-install test: mock missing bun, verify install script is invoked
- Verification step test: mock `beastmode --version` failure, verify error reporting

## Out of Scope

- `claude plugin add` as an alternative install path (future work)
- IDE detection / multi-IDE support (beastmode is Claude Code only)
- Project-level initialization (`/beastmode init` stays separate)
- Linux/Windows support (macOS only for now)
- Interactive prompts or fancy terminal UI in the installer
- Auto-update / background update checking
- Homebrew formula

## Further Notes

- The npm package name `beastmode` is available on npmjs.com as of 2026-04-06
- Claude-mem's installer was used as reference architecture — same pattern of npm-as-delivery-truck with manual plugin registration
- The `plugin/` directory in the npm package should be a build artifact (copy from repo root at publish time), not a source-of-truth duplication

## Deferred Ideas

- `claude plugin add` alternative install path with auto-CLI-bootstrap hook
- Pre-compiled binary distribution (removes bun runtime dependency)
- Homebrew formula for native macOS package management
- CI/CD-specific install mode with structured JSON output
- Version pinning / rollback support
