## Problem Statement

The "watch: running" status indicator in the dashboard header bar wraps to two lines when the NyanBanner component's `flexGrow={1}` claims too much horizontal space. This is a layout regression caused by a missing flex-shrink constraint on the status container — Yoga distributes compression equally across flex children, squeezing the status text below its natural width.

## Solution

Add `flexShrink={0}` to the status box container in ThreePanelLayout so Yoga never compresses it below its intrinsic width. As a secondary defense, add `wrap="truncate-end"` to the status Text element so it truncates gracefully on extremely narrow terminals instead of wrapping.

## User Stories

1. As a dashboard user, I want the "watch: running/stopped" indicator to always display on a single line, so that the header bar layout remains stable and readable.

## Implementation Decisions

- Add `flexShrink={0}` to the `<Box>` wrapping the watch status and version text in `ThreePanelLayout`
- Add `wrap="truncate-end"` to the watch status `<Text>` element as a secondary defense
- The NyanBanner already handles variable width gracefully via its dot-trail extension — it absorbs the flex pressure naturally
- No changes to NyanBanner, version text, or any other header elements

## Testing Decisions

- Manual visual verification in the dashboard at various terminal widths
- No automated tests — this is a pure layout property change with no behavioral logic

## Out of Scope

- Auditing other header elements for similar wrapping issues
- Automated layout regression testing for the TUI
