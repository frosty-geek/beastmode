## Problem Statement

The README describes a Claude Code plugin with slash commands. The product is now a CLI orchestrator with a fullscreen TUI dashboard that drives autonomous agents through parallel epics, with GitHub sync, liveness detection, and a self-improving knowledge hierarchy. New users have no way to understand the dashboard, compare Beastmode to alternatives (Superpowers, GSD, Beads), or discover the two-step install path (plugin + CLI). The README is a liability — it undersells the product and confuses the install.

## Solution

Incrementally update the existing README structure to reflect the current product. Add a Dashboard hero section with terminal recording placeholder, a dedicated competitor comparison table (generous and honest), a two-step install flow, and updated descriptions. Keep existing sections that still work (Why?, SVGs, What Beastmode Is NOT). Target 250+ lines for full treatment.

## User Stories

1. As a Claude Code power user, I want a README that explains why Beastmode exists, so that I can decide if it solves my problem in under 30 seconds.
2. As a new user, I want to see the Dashboard in action (terminal recording), so that I understand the autonomous pipeline without reading paragraphs of text.
3. As a tool shopper, I want an honest comparison table (GSD, Beads, Superpowers, Beastmode), so that I pick the right tool for my workflow without researching each one separately.
4. As a new user, I want clear two-step install instructions (plugin + CLI), so that I get both the skills and the dashboard working on first try.
5. As a developer evaluating Beastmode, I want to understand the two modes of operation (manual slash commands vs. autonomous dashboard), so that I know the product scales from supervised to fully autonomous.
6. As a returning user, I want the README to reflect the current feature set (dashboard, GitHub sync, parallel epics, liveness detection), so that I can point others to accurate documentation.
7. As a contributor, I want the README structure to be stable and accurate, so that PRs don't need to fix misleading claims.
8. As a developer who needs quick fixes, I want to know that manual slash commands still work, so that I don't think the dashboard is mandatory for simple tasks.

## Implementation Decisions

- **Structure**: Incremental update to existing README. Keep the bones (banner, tagline, What It Does, How It Works, What's Different, Why?, What Beastmode Is NOT, Credits, License). Add new sections: Dashboard hero, Comparison, two-step Install.
- **Dashboard emphasis**: Dashboard-first narrative. The autonomous pipeline is the headline feature. Manual slash commands become "also works manually" — not the primary story.
- **Dashboard visual**: Terminal recording (GIF/SVG) embedded after the Dashboard section heading. The recording itself is a deferred deliverable — the PRD specs a placeholder with instructions for future recording.
- **Comparison section**: Dedicated section with table comparing GSD, Beads, Superpowers, and Beastmode across dimensions: primary value, ceremony level, phase structure, context persistence, dashboard/TUI, pipeline orchestration, self-improving, multi-epic parallel, target user. Tone is generous — recommend the right tool for the job, not "Beastmode wins everything."
- **Install flow**: Two-step. Step 1: Plugin install (`claude plugin add beastmode@beastmode-marketplace`). Step 2: CLI install for dashboard (`cd cli && bun install && bun link`). Both clearly labeled with what each gives you.
- **Audience**: Claude Code power users. No "what is Claude Code" explainer. Assume tool familiarity.
- **Existing content kept**: Why? section (SAFe argument), existing SVG diagrams (progressive hierarchy, retro bubble-up, SAFe), What Beastmode Is NOT section.
- **Existing content dropped**: Deep-dive doc links (docs/progressive-hierarchy.md, docs/retro-loop.md, docs/configurable-gates.md) — remove the [Read the full argument] links.
- **Skills table update**: Update to reflect current skills and add the `beastmode` CLI commands (`beastmode dashboard`, `beastmode <phase> <slug>`, `beastmode compact`).
- **Length target**: 250+ lines. Full treatment.
- **What's Different update**: Update the three differentiators to reflect the dashboard/pipeline story. Add "Autonomous pipeline with a dashboard" as a fourth differentiator or replace the "Progressive autonomy" section with the dashboard narrative.
- **Research artifact**: Competitor research saved to `.beastmode/artifacts/research/2026-04-06-competitor-comparison.md` with sources from Perplexity.

## Further Notes

- The previous README rework PRD (2026-03-04-readme-rework.md) established good principles: problem-first hook, kill status tables, active voice throughout, under 160 lines. This PRD explicitly raises the line target to 250+ because the product scope has grown significantly (dashboard, CLI, comparison section).
- Beads' core feature (persistent memory) has been partially absorbed into Claude Code's native Tasks system (v2.1.16+). The comparison should note this.
- GSD works across Claude Code, OpenCode, Gemini CLI, and Codex — Beastmode is Claude Code only. The comparison should be honest about this.
