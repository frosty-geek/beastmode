## Problem Statement

Dashboard session statistics — totals, success rates, phase durations, re-dispatches — are held exclusively in memory by the `SessionStatsAccumulator`. Every dashboard restart resets all metrics to zero, making it impossible to track pipeline health over time.

## Solution

Persist session stats to a global JSON file in `.beastmode/state/`. On startup, load the persisted stats as an "all-time" view while maintaining a separate fresh accumulator for the current session. The DetailsPanel gains a keyboard toggle to switch between all-time (default) and current-session views. Stats flush to disk on every `session-completed` event for maximum durability.

## User Stories

1. As a pipeline operator, I want to see cumulative session statistics across dashboard restarts, so that I can track pipeline health over days and weeks.
2. As a pipeline operator, I want to toggle between all-time and current-session stats in the dashboard, so that I can distinguish long-term trends from the current run.
3. As a pipeline operator, I want stats to persist on every session completion, so that a crash or kill doesn't lose data from the current session.
4. As a pipeline operator, I want the dashboard to start cleanly even if the stats file is missing or corrupt, so that I never get blocked by stale state.
5. As a pipeline operator, I want to reset all-time stats by deleting a single file, so that I don't need a special command or UI for a rare operation.

## Implementation Decisions

- **Storage**: Single global file at `.beastmode/state/dashboard-stats.json` (gitignored with existing state/ patterns)
- **Flush trigger**: Write on every `session-completed` event. File is small (~1KB), I/O cost is negligible
- **Schema**: Summary stats with counts — no raw duration arrays. Phase averages are updated incrementally via `(oldAvg * oldCount + newVal) / (oldCount + 1)`. Schema includes:
  - `total`, `successes`, `failures`, `reDispatches`, `cumulativeMs` (scalar counters)
  - `phaseDurations`: `{ [phase]: { avgMs: number, count: number } }` (incremental averages)
  - `completedKeys`: `string[]` (persisted for cross-session re-dispatch detection)
  - `schemaVersion`: integer for forward compatibility
- **Load behavior**: On startup, load persisted file as historical stats. Create a fresh `SessionStatsAccumulator` for the current session. Two independent stat sets, never merged in memory
- **Display**: DetailsPanel defaults to all-time view. A keyboard shortcut toggles to current-session view. Only one view visible at a time
- **Accumulator changes**: Add a persistence callback or separate persistence module that subscribes to the same events and updates the file. The existing `SessionStatsAccumulator` stays focused on in-memory accumulation
- **Graceful degradation**: Missing file → empty historical stats. Corrupt/unparseable file → warn to log, treat as missing. No blocking errors
- **Reset**: Manual deletion of `dashboard-stats.json`. No UI or CLI command

## Testing Decisions

- Unit test the persistence module: serialize → write → read → deserialize round-trip
- Unit test incremental average computation: verify `(oldAvg * oldCount + newVal) / (oldCount + 1)` produces correct rolling averages
- Unit test graceful degradation: missing file, corrupt JSON, schema version mismatch
- Unit test completedKeys persistence: verify re-dispatch detection works across simulated restarts
- Integration test: accumulator events → file written → new accumulator loads correct historical state
- Existing `session-stats.test.ts` patterns provide prior art for accumulator testing with mock EventEmitters

## Out of Scope

- Per-epic stats breakdown (global only)
- Lifecycle log persistence (stats only)
- Dashboard reset shortcut (manual file deletion)
- Stats export or reporting commands
- Stats aggregation across multiple machines

## Further Notes

The `completedKeys` array grows with unique `epic:phase:feature` combinations but is practically bounded by project size. For very long-running projects, a future enhancement could prune keys for completed/cancelled epics.
