# Release: bdd-loop

**Bump:** minor
**Date:** 2026-04-05

## Highlights

Adds BDD-driven validation feedback loop: inline Gherkin scenarios in plan templates, BDD verification escalation in implement, and targeted re-dispatch of failed features from validate back through the pipeline.

## Features

- feat(plan-inline-gherkin): rewrite plan step 4 for batch invoke and inline distribution
- feat(plan-inline-gherkin): update integration tester agent input/output contract
- feat(plan-inline-gherkin): add integration test scenarios to feature plan template
- feat(implement-bdd-loop): add Task 0 and convention discovery to write-plan context
- feat(implement-bdd-loop): add Task 0 pattern to Write Plan step
- feat(implement-bdd-loop): add BDD verification section after wave loop
- feat(implement-bdd-loop): add BDD escalation docs, status summary stats, and L1 context
- feat(implement-bdd-loop): add BDD verification escalation documentation
- feat(validate-feedback): add ManifestFeature.reDispatchCount and regressFeatures() pure function
- feat(validate-feedback-loop): Add REGRESS_FEATURES event to epic machine
- feat(validate-feedback-loop): update existing features for targeted re-dispatch
- feat(validate-feedback-loop): wire failedFeatures through generate-output and reconcile
- feat(validate-feedback-loop): update validate skill with targeted re-dispatch behavior

## Fixes

- fix(plan-inline-gherkin): merge duplicate no-scenario bullets into single action
- fix(validate-feedback-loop): add file-permissions to test fixture, guard runner against missing config
